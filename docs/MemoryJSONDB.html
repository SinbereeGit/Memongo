MemoryJSONDB

MemoryJSONDB

A lightweight in-memory JSON database with MongoDB-like style API. The database keeps all data in memory and persists it through user-provided read/write functions.

Constructor

new MemoryJSONDB(getJSONObj, writeJSONObj)

Description:
  • Create a MemoryJSONDB instance.
Source:
  • memongo.js, line 597
Examples
// In Wechat Miniprogram, using local storage as the persistant method
const db = new MemoryJSONDB(
  async () => {
    try {
      return (await wx.getStorage({ key: "localDB" })).data;
    } catch {
      return {};
    }
  },
  async (jsonObj, resolve, reject) => {
    wx.setStorage({
      key: "localDB",
      data: jsonObj,
      success: resolve,
      fail: reject,
    });
  },
);
await db.init();

const users = await db.createCollection("users");
// The persistant storage stores the JSON string
const db = new MemoryJSONDB(
  async () => JSON.parse(localStorage.getItem("db")),
  async (data, resolve, reject) => {
    try {
      localStorage.setItem("db", JSON.stringify(data));
      resolve();
    } catch (err) {
      reject(err);
    }
  }
);

write() → {Promise.<void>}

Description:
  • This method implements write debouncing by: 1. Marking the database as dirty (needs write) 2. Waiting 200ms for potential additional changes 3. Writing to persistent storage only if still dirty
Source:
  • memongo.js, line 683
Examples
// Write with debouncing
await db.write();
console.log("Database persisted");
// Handle write errors
try {
  await db.write();
} catch (error) {
  console.error("Failed to save database:", error);
}
Throws:
When the storage write operation fails (e.g., storage quota exceeded)
Type
Error
Returns:
A promise that resolves when the write operation completes: - Resolves immediately if no changes were made during the debounce period - Resolves after successful storage write - Rejects if the storage write fails
Type
Promise.<void>
